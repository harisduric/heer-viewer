CONTEXT: The "Drucken" feature was implemented using an off-screen 
React portal (position: fixed; top: -9999px) with #root set to 
visibility: hidden during print. Result: print preview shows 7 pages 
instead of 4, and the canvas content is blank (only the "BO" title 
text appears, no drawing/overlay).

Likely causes:
- Canvases rendered far off-screen (-9999px) don't reliably paint in 
  some browsers
- visibility: hidden on #root still occupies layout space, causing 
  extra (blank) pages in the print output alongside the 4 portal pages

NEW APPROACH - capture on-screen canvases as images:
1. When "Drucken" is clicked, programmatically iterate through steps 
   BO, SE, KS, DE in the NORMAL (on-screen, visible) viewer - i.e. 
   briefly set the active step to each section in sequence, wait for 
   PdfViewer's onRendered callback to confirm the canvas has finished 
   drawing (including overlays), then call canvas.toDataURL('image/png') 
   to capture a snapshot image for that section.
2. After collecting all 4 images (BO, SE, KS, DE), build a print-only 
   view containing 4 pages, each showing one captured image, sized to 
   fit the page (reuse the same fit-to-page logic: portrait images like 
   KS should be centered and scaled to fit A4 landscape without 
   cutting off).
3. Use `display: none` (not visibility: hidden) on #root during print, 
   and render the print view as a sibling element that is normally 
   display:none and only display:block during @media print - this 
   avoids extra blank pages from hidden-but-space-occupying elements.
4. After printing (afterprint event), restore the viewer to whatever 
   step the user was on before clicking "Drucken".
5. While capturing images (steps 1-2), show "Wird gedruckt..." on the 
   Drucken button and briefly disable step navigation, since the view 
   will flicker through BO/SE/KS/DE.
