## 1. PROJECT OVERVIEW

**App name:** Heer Viewer  
**Purpose:** A production-floor web app used by B. Heer AG staff on tablets and PCs. It eliminates the need to manually compare paper schema drawings side-by-side. The user imports an execution description PDF; the app automatically finds the matching schema drawing, overlays all dimension values onto it, and displays each section step-by-step in a workflow.

**No PDF export needed.** The app IS the viewer вҖ” used live in production.

---

## 2. TECH STACK

- **Frontend:** React + Tailwind CSS
- **Backend:** Node.js + Express
- **Database:** PostgreSQL with Drizzle ORM (Replit built-in) вҖ” stores schema slot registry + coordinates
- **File Storage:** Replit Object Storage (GCS) вҖ” stores schema drawing PDFs persistently
- **PDF Text Extraction:** `pdf-parse` (npm)
- **PDF Rendering:** `pdfjs-dist` (PDF.js, canvas-based вҖ” NEVER iframe or embed)
- **Run command:** `npm run dev` via `concurrently` (frontend port 3000, backend port 3001)

## 5. FILE NAMING & MATCHING

The schema library contains **17 fixed, named slots** (without `.pdf` extension). PDFs are uploaded into slots via the Bibliothek admin page вҖ” the uploaded file can have any filename.

**Matching logic for execution description upload:**
1. Strip `.pdf` from the uploaded filename
2. Check if any of the 17 canonical slot names is contained as a substring (case-sensitive)
3. Sort names longest-first before checking to prevent false partial matches (important for `BASIC`)
4. Match found вҶ’ load schema drawing and navigate to Viewer
5. No match вҶ’ error: `"Keine passende Schemazeichnung gefunden fГјr '[filename]'"`

## 7. EXECUTION DESCRIPTION вҖ” PARSING RULES

### Format:
```
[SECTION] - [CODE] - [VALUE]
```
Separator is ` - ` (space-dash-space).

### Global dimensions (Page 1):
Lines like `IM - LГ„NGE - 990`, `AM - BREITE - 792`, `U_QUE - MIN. 580 - 626`
- Parse as key `IM-LГ„NGE`, value `990`
- Store in `globalDimensions` object

### Section L-codes (Page 2):
- Sections: `BO`, `SE`, `KS`, `DE`
- Codes: `L01` to `L20` вҶ’ normalize to `L1`вҖ“`L20` (strip leading zero)
- Values: preserve Swiss format exactly (e.g. `1'062`, `2'020`)

### Range values:
`KS - L03 - (500-1100) - 520` вҶ’ use only the **last value** `520`. Ignore the range in parentheses.

### Ignore completely:
- Lines with `LEER` code
- Any entry where value is `0`
- Separator lines `- - - - -`
- Metadata lines (`Nr.`, `SCHEMA`, `STANDARD`, `AusfГјhrungsbeschreibung`)

### ANO_CODE entries:
Format: `KS1 - ANO_CODE - Z01 - 11`
- If value is `0` вҶ’ ignore entirely (no Hebegurt step shown)
- If value is non-zero (e.g. `11`, `12`, `21`, `22`) вҶ’ store as `{ section: 'KS', optionCode: 'Z01', value: '11' }`
- Multiple ANO_CODEs can be active simultaneously

### Parsing function:
```javascript
function parseExecutionDescription(pdfText) {
  const lines = pdfText.split('\n').map(l => l.trim()).filter(Boolean);
  const result = {
    globalDimensions: {},
    sections: { BO: {}, SE: {}, KS: {}, DE: {} },
    anoCodes: []
  };
  const globalPrefixes = ['IM', 'AM', 'LM', 'U_QUE'];

  for (const line of lines) {
    if (line.startsWith('- - -') || line.startsWith('Nr.') ||
        line.startsWith('SCHEMA') || line.startsWith('STANDARD') ||
        line.includes('AusfГјhrungsbeschreibung')) continue;

    const parts = line.split(' - ').map(p => p.trim());
    if (parts.length < 3) continue;

    const [rawSection, code, ...rest] = parts;
    const section = rawSection.replace(/\d+$/, ''); // KS1 вҶ’ KS
    const rawValue = rest[rest.length - 1];

    // ANO_CODE
    if (code === 'ANO_CODE') {
      const optionCode = rest[0];
      if (rawValue && rawValue !== '0') {
        result.anoCodes.push({ section, optionCode, value: rawValue });
      }
      continue;
    }

    if (code === 'LEER' || !rawValue || rawValue === '0') continue;

    // Global dimensions
    if (globalPrefixes.some(p => rawSection.startsWith(p))) {
      const key = `${rawSection}-${code}`;
      result.globalDimensions[key] = rawValue;
      continue;
    }

    // Section L-codes
    if (['BO', 'SE', 'KS', 'DE'].includes(section) && /^L\d{2}$/.test(code)) {
      const label = `L${parseInt(code.replace('L', ''), 10)}`;
      result.sections[section][label] = rawValue;
    }
  }
  return result;
}
```

## 16. KNOWN FIXES

1. **Koordinaten Editor PDF Canvas вҖ” Scaling**
   ALWAYS use `ResizeObserver` on the container ref to calculate scale.
   NEVER use `window.innerWidth`.
   Formula: `scale = containerWidth / 595.276`

2. **PDF Cache вҖ” Koordinaten Editor**
   ALWAYS add `?t=Date.now()` to all PDF fetch URLs in the Koordinaten Editor
   to prevent stale cached PDFs from being displayed.

---

## 17. PERMANENT FIXES

These rules must NEVER be reverted. Any future edit to the Koordinaten Editor must preserve them.

1. **PDF Canvas Scaling вҖ” ResizeObserver**
   ALWAYS derive `renderScale` from a `ResizeObserver` on the container `div` ref.
   NEVER read `window.innerWidth` or `clientWidth` at render time.
   Formula applied at every resize: `scale = containerWidth / vpNat.width`
   (where `vpNat.width` is the PDF's natural width at scale 1.0, вүҲ 595.276 pt for A4)

2. **PDF Fetch Cache-Bust**

### Auto-detection (runs on every Bibliothek upload)

When a schema PDF is uploaded:
- pdf-parse runs its pagerender hook on page 2
- Every text item's transform[4] = X, transform[5] = Y (PDF space, bottom-up)
- Y is converted to screen-space: y = pageHeight - transform[5]
- Items where transform is not a 6-element numeric array are skipped
- Section headers BO / SE / KS / DE are found by text match (largest font wins)
- All labels matching /^L\d+$/ are assigned to the nearest section header
  by Euclidean distance (fallback centres used if header not found in PDF)
- Results saved as page2: { BO: {L1:{x,y},...}, SE:..., KS:..., DE:... }
- Admin can re-trigger at any time: POST /api/schema/:name/redetect
